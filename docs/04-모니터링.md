# 모니터링

> 상위 문서: [`../PRD.md`](../PRD.md) §5.5

## 개요

담당자가 주문·재고 현황을 한눈에 파악할 수 있도록 상태별 집계 정보를 제공하는 조회 전용(읽기 전용) 기능. 데이터를 변경하지 않는다.

## 하위 기능

### 1) 주문량 확인
- 현재 상태별 주문 수를 집계해 표시한다.
  - `RESERVED` 건수
  - `CONFIRMED` 건수
  - `PRODUCING` 건수
  - `RELEASE` 건수
- `REJECTED` 상태 주문은 정상 흐름 밖의 상태이므로 **집계에서 제외**한다.

### 2) 재고량 확인
- 시료별 현재 재고 수량을 표시한다.
- 각 시료에 대해 "주문 대비 재고 수량"에 따른 상태를 함께 표기한다.

| 표기 | 조건 |
|---|---|
| 여유 | 재고 수량이 해당 시료에 대한 미출고(대기) 주문 수량 합계보다 충분히 많음 |
| 부족 | 재고 수량이 대기 주문 수량 합계보다 적음(단, 0은 아님) |
| 고갈 | 재고 수량이 0 |

> "여유/부족" 판정 기준(예: 재고 ≥ 대기 주문 수량 합계 → 여유, 그 반대는 부족)은 구현 시 정확한 임계값을 확정한다. PDF 명세에는 세 단계 구분만 명시되어 있고 수치 기준은 없으므로, 아래를 기본 규칙으로 제안한다.
> - 고갈: 재고 수량 == 0
> - 부족: 0 < 재고 수량 < 미출고 대기 주문 수량 합계
> - 여유: 재고 수량 ≥ 미출고 대기 주문 수량 합계

## 표시 항목 예시

| 시료 ID | 이름 | 현재 재고 | 대기 주문 수량 합계 | 재고 상태 |
|---|---|---|---|---|
| S001 | ... | 12 | 5 | 여유 |
| S002 | ... | 2 | 8 | 부족 |
| S003 | ... | 0 | 3 | 고갈 |

## 예외/유효성 처리

- 등록된 시료/주문이 없는 경우 각각 "데이터 없음"을 표시한다.
- 모니터링 화면은 조회 전용이며 어떠한 입력도 상태를 변경하지 않는다.
