# 주문관리 (접수 / 승인 / 거절)

> 상위 문서: [`../PRD.md`](../PRD.md) §5.3, §5.4, §4.2

## 개요

고객의 시료 요청을 접수(RESERVED)하고, 생산라인 담당자가 이를 승인 또는 거절하는 기능. 승인 시 재고 상황에 따라 즉시 출고 대기(CONFIRMED) 또는 생산 대기(PRODUCING)로 자동 분기된다.

## 주문 속성

| 속성 | 필수 | 설명 |
|---|---|---|
| 주문 ID | 시스템 자동 채번 | 주문 식별자 |
| 시료 ID | Y | 주문 대상 시료. 반드시 등록된 시료여야 함 |
| 고객명 | Y | 주문한 고객 이름 |
| 주문 수량 | Y | 1 이상의 정수 |
| 상태 | 시스템 관리 | RESERVED / REJECTED / PRODUCING / CONFIRMED / RELEASE |
| 접수 일시 등 | 선택 | 모니터링/큐 정렬(FIFO)에 필요하면 기록 |

## 하위 기능

### 1) 시료 주문 (접수)
- **입력값**: 시료 ID, 고객명, 주문 수량
- **처리**:
  - 시료 ID가 등록되지 않은 경우 주문 생성을 거부한다.
  - 주문 수량은 1 이상이어야 한다.
  - 신규 주문은 `RESERVED` 상태로 생성된다.
- **결과**: 주문이 "접수된 주문 목록"에 추가되어 승인/거절 대상이 된다.

### 2) 접수된 주문 목록 확인
- `RESERVED` 상태의 주문만 목록으로 표시한다.
- 표시 항목(예): 주문 ID, 시료명, 고객명, 주문 수량

### 3) 주문 승인
- 대상: `RESERVED` 상태의 특정 주문
- **처리 로직**:
  1. 승인 대상 주문의 시료 ID 기준으로 현재 재고 수량을 확인한다.
  2. **재고 ≥ 주문 수량** → 재고에서 주문 수량만큼 즉시 차감하고, 주문 상태를 `CONFIRMED`로 전환한다.
  3. **재고 < 주문 수량** → 남아있는 재고를 **0이 될 때까지 전부** 이 주문에 차감(할당)하고, 부족분(`주문 수량 - 승인 시점 재고`)만큼 생산이 필요하다고 판단해 생산라인 큐에 자동 등록하고, 주문 상태를 `PRODUCING`으로 전환한다. (생산 로직 상세는 [`06-생산라인.md`](06-생산라인.md) 참고)
- **재고 확인/차감은 이 승인 시점에만 이루어진다.** 이후 생산이 완료되어도 재고 확인 로직이 재실행되지 않고, 생산 결과(부족분 충족 + 초과분 재고 가산)만 반영된다.
- **동일 시료에 대한 후속 주문 처리**: 위 정책 덕분에 같은 시료의 다음 주문을 승인할 때는 이미 앞 주문에 의해 줄어든(또는 0이 된) 재고를 기준으로 판단하게 되어, 앞 주문에 할당된 몫과 절대 겹치지 않는다. 별도의 "예약된 재고 제외" 로직을 추가로 둘 필요는 없다 — 재고를 승인 시점에 즉시 차감하는 것 자체가 이 요구사항을 만족시킨다.
  - 예: 재고 10에서 주문A(수량 15) 승인 → 재고 10 전부 A에 할당, 재고는 0, 부족분 5는 생산 큐에 등록. 뒤이어 주문B(같은 시료, 수량 8) 승인 시 재고는 이미 0이므로 부족분 8 전체가 새로 생산 큐에 등록된다(A의 생산 여부와 무관).
- 이미 `RESERVED`가 아닌 주문(예: 이미 승인/거절된 주문)에 대한 재승인 시도는 거부한다.

### 4) 주문 거절
- 대상: `RESERVED` 상태의 특정 주문
- **처리**: 즉시 `REJECTED` 상태로 전환한다. 이후 이 주문은 정상 흐름에서 제외되며 모니터링 집계 대상에서도 빠진다.
- `RESERVED`가 아닌 주문은 거절할 수 없다.

## 상태 전이 요약

```
(신규) --접수--> RESERVED
RESERVED --승인(재고충분)--> CONFIRMED
RESERVED --승인(재고부족)--> PRODUCING --생산완료--> CONFIRMED
RESERVED --거절--> REJECTED
```

## 예외/유효성 처리

| 상황 | 처리 |
|---|---|
| 미등록 시료 ID로 주문 접수 시도 | 거부, 에러 메시지 |
| 주문 수량이 0 이하 | 거부, 에러 메시지 |
| `RESERVED`가 아닌 주문에 승인/거절 시도 | 거부, 에러 메시지 |
| 존재하지 않는 주문 ID 지정 | 거부, 에러 메시지 |
