# 출고처리

> 상위 문서: [`../PRD.md`](../PRD.md) §5.6

## 개요

재고가 충분해져 `CONFIRMED` 상태가 된 주문에 대해 실제 출고를 실행하는 기능. 출고가 완료되면 주문은 시스템상 최종 상태인 `RELEASE`로 전환된다.

## 하위 기능

### 1) 출고 대상 주문 확인
- `CONFIRMED` 상태의 주문 목록을 표시한다.
- 표시 항목(예): 주문 ID, 시료명, 고객명, 주문 수량

### 2) 출고 실행
- **입력값**: 출고할 주문 ID
- **처리**:
  - 대상 주문이 `CONFIRMED` 상태인지 확인한다. 아니라면 거부한다.
  - 주문 상태를 `RELEASE`로 전환한다.
- **재고 반영**: 재고 차감은 승인 시점(§ [`03-주문관리.md`](03-주문관리.md) 주문 승인 로직)에 이미 이루어졌으므로, 출고 처리 시점에 재고를 추가로 차감하지 않는다. (재고 차감 시점은 구현 시 "승인 시" 또는 "출고 시" 중 하나로 명확히 확정하고 일관되게 적용할 것 — 아래 참고)

> **구현 시 확정 필요**: PDF 명세는 "재고가 충분해진 CONFIRMED 주문에 대해 출고를 처리"라고만 서술한다. 재고 차감 시점을 (a) 주문 승인 시 즉시 차감, (b) 출고 실행 시 차감 중 하나로 정해야 하며, 모니터링의 재고 수치와 일관되게 한 가지 정책만 사용해야 한다. 본 문서는 (a) 승인 시 차감을 기본안으로 제안한다.

## 예외/유효성 처리

| 상황 | 처리 |
|---|---|
| `CONFIRMED`가 아닌 주문에 출고 시도 | 거부, 에러 메시지 |
| 존재하지 않는 주문 ID 지정 | 거부, 에러 메시지 |
| 이미 `RELEASE`된 주문에 재출고 시도 | 거부, 에러 메시지 |
