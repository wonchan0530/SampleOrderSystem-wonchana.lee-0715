# 시료관리

> 상위 문서: [`../PRD.md`](../PRD.md) §5.2, §4.1

## 개요

시료(Sample)는 시스템의 가장 기본이 되는 단위다. 시스템에 등록된 시료만 주문할 수 있으므로, 시료관리 기능은 다른 모든 기능(주문, 생산, 출고, 모니터링)의 전제 조건이다.

## 시료 속성

| 속성 | 타입(예상) | 필수 | 설명 |
|---|---|---|---|
| 시료 ID | 문자열/숫자 | Y | 시료를 식별하는 고유 값. 중복 등록 불가 |
| 이름 | 문자열 | Y | 시료 이름. 검색 키로 사용됨 |
| 평균 생산시간 | 숫자 | Y | 시료 1개 생산에 걸리는 평균 시간(단위는 구현 시 확정, 예: 분) |
| 수율 | 숫자(0~1) | Y | 정상 생산 시료 수 / 총 생산 시료 수. 예: 0.9 |
| 현재 재고 수량 | 숫자 | 시스템 관리 | 사용자가 직접 입력하지 않고, 생산/출고에 따라 시스템이 관리하는 파생 값 (등록 시 초기값 0) |

## 하위 기능

### 1) 시료 등록
- **입력값**: 시료 ID, 이름, 평균 생산시간, 수율
- **처리**:
  - 동일한 시료 ID가 이미 존재하면 등록을 거부하고 에러 메시지를 표시한다.
  - 수율은 0 초과 1 이하 범위여야 한다(0이면 정상품이 전혀 생산되지 않아 무한 생산 루프가 발생하므로 유효성 검사 필요).
  - 평균 생산시간은 0보다 커야 한다.
  - 등록 시 초기 재고 수량은 0으로 설정한다.
- **결과**: 신규 시료가 시스템에 추가되고, 이후 해당 시료 ID로 주문이 가능해진다.

### 2) 시료 조회
- 등록된 모든 시료 목록을 표시한다.
- 각 시료의 현재 재고 수량을 함께 표시한다.
- 목록이 비어있는 경우 "등록된 시료 없음" 등 안내 메시지를 표시한다.

### 3) 시료 검색
- 이름 등 속성으로 특정 시료를 검색한다.
- 부분 일치(포함 검색) 지원 여부는 구현 시 결정하되, 최소한 완전 일치 검색은 지원해야 한다.
- 검색 결과가 없으면 "일치하는 시료 없음"을 표시한다.

## 예외/유효성 처리

| 상황 | 처리 |
|---|---|
| 중복된 시료 ID로 등록 시도 | 등록 거부, 에러 메시지 표시 |
| 수율이 0 이하이거나 1 초과 | 등록 거부, 에러 메시지 표시 |
| 평균 생산시간이 0 이하 | 등록 거부, 에러 메시지 표시 |
| 미등록 시료 ID로 주문 시도 (주문관리 쪽 로직) | 주문 생성 거부 — [`03-주문관리.md`](03-주문관리.md) 참고 |

## 데이터 영속성

- 시료 데이터는 애플리케이션 재실행 후에도 유지되어야 한다.
- 재고 수량은 주문 승인(생산 완료 시)·출고 처리에 따라 변동되므로, 시료 데이터와 함께 영속화되어야 한다.
